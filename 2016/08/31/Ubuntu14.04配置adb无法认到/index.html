Ubuntu14.04配置adb无法认到

发表于 2016-08-31   |   分类于 Ubuntu   |  

现象：

今天重装了Ubuntu系统，升级到了14.04版本，没想到安装完adt bundle之后配置adb，竟然识别不到了，开始以为环境变量配置不对，后来发现在platform-tools下直接调用adb竟然也出现：

bash: ./adb: No such file or directory 

无奈，去网上搜索，终于弄明白是因为64位的Linux，没有安装32位的依赖库导致的，需要重新安装下就好了：

$ sudo apt-get install ia32-libs

但此时另外一个问题出现了，由于我安装的是Ubuntu14.04版本，发现安装ia32-libs无法安装；

搜索后网上说的解决方案大体如下：

sudo -i //切换到root权限

cd /etc/apt/sources.list.d //进入apt源列表

echo "deb http://archive.ubuntu.com/ubuntu/ raring main restricted universe multiverse" > ia32-libs-raring.list //添加ubuntu 13.04的源，因为13.10的后续版本废弃了ia32-libs

apt-get update //更新源

apt-get install ia32-libs //安装ia32-libs(我在做到此处时ia32-libs依旧无法正常安装，故我没有采用这种方法)

rm ia32-libs-raring.list

apt-get update //恢复源

解决方法：

我所采取的方法是：

sudo apt-get install libgtk2.0-0:i386

此时仍会出现：

$ ./adb

./adb: error while loading shared libraries: libstdc++.so.6: cannot open shared object file: No such file or directory

再执行命令：

sudo apt-get install lib32stdc++6

然后：

$ adb version

Android Debug Bridge version 1.0.31

问题成功解决;
